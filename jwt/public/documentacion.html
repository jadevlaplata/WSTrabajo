ws_calendar

General notes

Este servicio está alojado en un PaaS Heroku. Puede ser consumido proporcionando un documento, nro de hash y un identificador de rol (enpoint permission). Se retorna un token que será utilizado como parámetro (token) en la cabecera para el resto de las peticiones.

API detail

permission

Retorna un token para generar una petición al enpoint timeForV.

  • HTTP

"idRol":54
